# 总结 
让我们列出一些要记住的模块关键点：
+ 模块有助于扩展FreeRADIUS的功能。
+ 一个模块可以通过使用命名部分来运行不同的实例。
+ 模块通过FreeRADIUS配置目录中modules子目录下的文本文件进行配置。
+ 部分内的模块顺序非常重要。
+ 模块具有影响请求流的返回代码。
+ 我们可以使用unlang来测试模块的特定返回码。
+ 某些模块可能会丢失，因为它们是单独包装的，必须先单独安装才能使用。

下一章将介绍EAP。 如果要在LAN上实施802.1x安全性或在Wi-Fi网络上实施WPA2 企业安全性，则需要了解EAP。
